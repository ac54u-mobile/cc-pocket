What's new in 1.5.0

New
• OpenCode support: a third agent alongside Claude Code and Codex — sessions, permissions and model selection all wired up
• Switch sessions across projects: a session chip beside the composer jumps straight to a session in another project, instead of backing out through the list
• Context usage moved into the composer: the debug-looking badge that floated over the message stream is retired, replaced by a steady readout next to the input — tappable for detail, and the first thing to yield when space is tight
• Per-model context window: a window belongs to the model, not to the phone — set the real size for each model right in the session info sheet, review and clear them in Settings
• Gateway model list now comes from the gateway itself: no more relying on a built-in table of vendor ids — you see what your gateway actually serves, and vendor generation changes follow automatically
• Desktop "Open Folder…": pick a project with the native directory chooser instead of typing a path

Fixed
• A context-window size you typed is no longer overridden by automatic inference
• Switching sessions lands on the latest message reliably, even after scrolling back through history
• Gateway model presets corrected: two retired ids removed, one updated to the current generation
• Fixed older app versions showing OpenCode session rows that couldn't be opened
