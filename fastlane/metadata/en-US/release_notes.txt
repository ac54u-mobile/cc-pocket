What's new in 1.4.0

New
• Open a project: browse your computer's folders from the phone and start a fresh session in any of them — no longer limited to resuming past sessions
• Faster model switching: a model chip next to the composer jumps straight to the model picker (phone and desktop)
• Embedded terminal on desktop: ⌘J toggles a docked terminal panel, sessions can open "inside the app", and the default is configurable in Settings
• macOS menu-bar status capsule: an always-on icon shows connection and turn state at a glance (five states), with a quick panel on click
• Rename sessions on desktop: right-click a session in the sidebar to rename inline — kept in sync with Claude Code's own records
• Bridge credentials (preview): connect Claude Code to external bots with least-privilege access (a Feishu group adapter ships as a reference), monitored and revocable from phone and desktop

Fixes & improvements
• First pairing is now resilient: if the very first connection got interrupted, the app could stay unable to connect forever (even re-scanning didn't help, only restarting the daemon did) — it now heals itself on the next reconnect
• iOS push fixed: some installs weren't receiving completion/approval notifications
• Links in messages no longer swallow a trailing bracket into the URL
• In bypassPermissions mode, plan confirmations are no longer auto-approved — they still wait for you

Note: some features need the computer-side cc-pocket-daemon on 1.4.0 — run "cc-pocket-daemon update" (or brew upgrade / scoop update).
