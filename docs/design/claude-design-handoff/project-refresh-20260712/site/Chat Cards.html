Backoff timer lives on the socket scope

The reconnect backoff is scheduled on the socket's own scope (WsClient.socketScope). When the socket closes after the 3rd failure that scope is cancelled — taking the pending retry with it, so a 4th attempt is never scheduled.

Suggested fix

  • Hoist retry scheduling to appScope so it survives socket teardown.
  • Add a test that force-closes after 3 retries and asserts a 4th fires.

Relevant files: WsClient.rs, Backoff.rs, reconnect_test.rs.
